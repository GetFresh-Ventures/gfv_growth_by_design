# Weekly Pipeline Review Workflow

**When**: Friday afternoon (or your preferred end-of-week time)
**Duration**: 30-45 minutes
**Skills used**: [deal-review](../skills/deal-review/SKILL.md), [pipeline-pulse](../skills/pipeline-pulse/SKILL.md)

---

## Steps

### 1. Pull Current Pipeline (5 min)
```
Pull all active deals from CRM. Use the deal-review skill to assess
health status for each deal.
```

### 2. Flag Issues (10 min)
Review each deal and flag:
- 🔴 **Stale deals** — no activity 14+ days
- 🔴 **Past close date** — overdue deals still marked active
- 🟡 **No next step** — deals without a defined next action
- 🟡 **Amount seems off** — unrealistic for current stage

### 3. Generate Pipeline Pulse (5 min)
```
Generate this week's pipeline pulse. Compare against last week's
numbers from brain/pipeline.md.
```

### 4. Decide Actions (10 min)
For each flagged deal, decide:
- **Re-engage**: Draft a follow-up email (use email-composer)
- **Progress**: Schedule next step in CRM
- **Close**: Mark as lost if dead (be honest)
- **Escalate**: Bring to team standup if needs group input

### 5. Update Brain (5 min)
```
Update brain/pipeline.md with this week's state.
Log any observations in brain/learnings.md.
```

### 6. Plan Next Week (5 min)
Identify the 3 most important deals to focus on next week.
Schedule any needed prep (meeting-prep skill for upcoming calls).

---

## What Good Looks Like
After this workflow:
- [ ] Every active deal has a defined next step
- [ ] No deals are stale without a plan
- [ ] Pipeline.md is current
- [ ] Top 3 priorities for next week are clear
- [ ] Any follow-up emails are drafted (awaiting approval)
